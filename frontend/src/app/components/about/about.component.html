UrbanPulse Architecture

A multi-agent AI system designed for real-world smart city incident management, built with Angular, Spring Boot, and LangGraph.

What is UrbanPulse?

UrbanPulse is a smart city incident management platform that connects citizens, municipal departments, and an autonomous AI agent pipeline. Citizens submit incident reports via a web interface; behind the scenes, three specialized AI agents classify, route, and monitor every case — without manual intervention.

The platform is designed as an academic showcase of multi-agent orchestration using LangGraph, demonstrating how frameworks like CrewAI, AutoGen, and MCP can be applied to real urban governance challenges.

The Three-Agent AI System

Each agent has a clearly scoped responsibility within the LangGraph stateful pipeline.

Agent 01

Classifier Agent

Analyses the free-text description, location data, and historical incident patterns to assign a category and a 1–5 priority score. Uses an LLM with structured JSON output schema to ensure consistent, machine-readable decisions.

LangChain OpenAI API Structured Output
Agent 02

Planner Agent

Receives the classified report and queries current department workloads and active incidents in the district. Decides the optimal assignment, sets an SLA deadline, and outputs a recommended action plan for the assigned team.

Resource Optimization SLA Management
Agent 03

Monitor Agent

Runs on a 15-minute cron cycle, polling all open incidents for SLA breaches. Generates escalation alerts, updates statuses automatically, and produces nightly summary reports — a fully autonomous background process.

Cron Scheduler SLA Monitoring Auto-Escalation

AI & Backend Integration

How the Magic Happens (Step-by-Step)

Step 1: Submission & Backend Handshake

Data Validation & Storage

When you submit a report, the Angular interface securely transmits it to our Spring Boot backend. The backend immediately stores your report in a MySQL database safely, assigns it a Tracking ID (Like #12), and sends a Live WebSocket Event to all connected online users to update their maps and dashboards instantly.

Step 2: AI Pipeline Trigger

The Multi-Agent Assessment

Less than a second after the report is saved, the backend awakens the Python AI Service. Three specialized Artificial Intelligence agents read your report's text and assess its context:

  • Classifier Agent: Determines if this is a "Traffic Accident" or "Power Outage," giving it an urgency score from 1 to 5. It uses LIVE tools like searching for nearby critical infrastructure.
  • Planner Agent: Analyses the Classifier's score, finds the exact department responsible (e.g., "Water & Sewage Directorate"), and sets a strict deadline (SLA) to resolve the issue.
  • Monitor Agent: Produces a highly confident final summary logic for the Dashboard.
Step 3: Autonomous Action

Dynamic Updates & Notifications

Once the AI concludes its assessment, it sends the parsed decision back to the Spring Boot backend via an encrypted internal bridge. The backend then:

  1. Dynamically changes the status from Pending to AI Processed to In Progress.
  2. Pushes real-time updates to the web dashboard so admins can see exactly which tools the AI used.
  3. Sends Push Notifications to phones and emails to the reporting citizens automatically.
